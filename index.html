P2T Inference Engine

A modular deep learning pipeline designed to convert raw handwriting into concise digital summaries. Utilizes a fine-tuned mBART model to contextually repair spelling errors and T5 for abstractive summarization.

Python PyTorch Transformers CNN
Source Code

The Problem: Noisy Handwriting

Standard Optical Character Recognition (OCR) systems struggle significantly with "noisy" input—messy handwriting, scribbles, paper stains, or poor lighting. When these raw OCR outputs are fed into summarization models, the result is often incoherent gibberish.

The Pipeline Solution

We engineered a multi-stage deep learning pipeline to bridge the gap between Vision and NLP:

1. Preprocessing: Raw images undergo adaptive thresholding and noise reduction.
2. Extraction (CNN): A CNN extracts visual features and converts them into a raw text sequence.
3. Repair (mBART): A fine-tuned mBART model acts as a "Correction Module," using context clues to fix spelling errors.
4. Summarization (T5): Finally, a T5 Transformer generates a concise abstractive summary.

Technical Milestones

  • Model Fine-Tuning: Trained mBART on a custom dataset of handwriting errors.
  • WER Reduction: Achieved a 15% reduction in Word Error Rate compared to using standard OCR tools alone.
  • Full-Stack Deployment: Wrapped the Python inference engine in a Flask API and connected it to a React frontend.

Disaster Management Hub

Developed for Bluehacks, this platform serves as a real-time command center for calamity response. It integrates Google Maps API to visualize hazard zones and rescue teams.

JavaScript Firebase Google Maps API
Source Code

The Hackathon Challenge

Built during the high-pressure environment of Bluehacks, the goal was to create a solution for rapid calamity response. In disaster scenarios, milliseconds matter, and traditional web pages that require manual refreshing are too slow.

Real-Time Architecture

The core of this application is Firebase Realtime Database. We chose this over a traditional SQL database to leverage its WebSocket-based synchronization. When a user reports a flood, the data is pushed to all connected Admin Dashboards instantly.

Key Features & Roles

  • Geospatial Plotting: Integrated Google Maps API to plot incidents using exact GPS coordinates.
  • Hazard Heatmaps: The system automatically aggregates data to visualize high-density danger zones (Red Zones).
  • Role-Based Views: Designed distinct interfaces for the Public (reporting incidents) and Admins (dispatching rescue teams).

Baranggay Info System

A centralized government record management system designed for LGUs. It streamlines resident data tracking, automates certificate issuance, and manages blotter reports.

PHP MySQL Bootstrap
Source Code

Digital Transformation

Many Local Government Units (LGUs) still rely on physical logbooks and paper files. This legacy method leads to data loss and slow retrieval times. This system digitizes the entire workflow of a Barangay hall.

Database & Backend Logic

The system is built on a Normalized MySQL Database to ensure data integrity. I implemented complex SQL queries with `JOINs` to handle intricate resident relationships (e.g., linking individuals to households and blotter cases). The backend is powered by PHP, implementing secure session management.

System Capabilities

  • Automated Documents: Generates official PDF documents (Barangay Clearance, Indigency, Residency) dynamically using stored resident data.
  • Full CRUD Management: Allows authorized staff to Create, Read, Update, and Delete records for residents and households.
  • Search & Analytics: Features a live search algorithm to find residents instantly and a dashboard that visualizes population demographics.
